## Configuration

The generated `<app-name>-config.toml` includes sections based on selected features:

```toml
[log]
level = "info"
format = "text"

[server]           # when API enabled
host = "0.0.0.0"
port = 8080

[database]         # when DB enabled
host = "localhost"
port = 5432
# ... driver-specific fields

[redis]            # when Cache=Redis
host = "localhost"
port = 6379
```

Configuration sources are loaded in this order (first wins):

1. Command-line flags
2. Environment variables
3. TOML config file
4. Default values

## Build Tasks

The generated `Taskfile.yml` includes:

| Task | Description |
|------|-------------|
| `task build` | Cross-compile for linux/darwin amd64+arm64 with CGO_ENABLED=0 |
| `task build-local` | Build for the local platform |
| `task test` | Run `go test ./...` |
| `task lint` | Run `golangci-lint` |
| `task frontend-build` | Build frontend assets with bun (UI feature only) |

## Running the Generated App

```sh
cd my-service

# Start the service
go run . serve

# Initialize a default config file
go run . init

# Show help
go run . --help

# Generate shell completions
go run . completion --shell bash
```

## Architecture Pattern

The generated API code follows a **Handler -> Service -> Storage** layering pattern:

- **Handler** -- HTTP request/response handling, input validation, calls service
- **Service** -- business logic, orchestration, calls storage
- **Storage** -- data access (in-memory by default, replace with DB calls)

## DNS SRV Resolution

When DB, Cache, or API is enabled, the generated `internal/resolve/` package provides transparent DNS SRV resolution. Service initialization code calls `resolve.LookupSRV()` before connecting, so configured hostnames that are SRV records are automatically resolved to real addresses.

This is useful with service discovery systems like Consul. If the hostname is not an SRV record, it passes through unchanged.

## Test Coverage

Every generated `.go` source file (except `main.go` and `embed.go`) has a corresponding `_test.go` file with at least one test function. Tests pass out of the box:

```sh
go test ./...
```
